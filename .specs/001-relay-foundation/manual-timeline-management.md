# 手动时间线管理

## 需求

- 管理员在后台手动创建模型上架、模型下架、倍率调整和服务变更等时间线记录。
- 业务操作本身不得自动生成时间线记录。
- 每条记录可保存为草稿或手动发布；用户端只返回已发布记录。
- 兼容已有公告数据：未携带 `published` 字段的历史记录继续视为已发布。

## 实现

- 继续使用 `console_setting.announcements` 保存时间线，避免引入重复的数据源。
- 内容设置中的公告入口显示为“时间线管理”。
- 管理页支持新增、编辑、删除、批量删除、发布状态切换和全局启停。
- `/api/status` 仅通过 `GetAnnouncements` 返回已发布记录。

## 验证

- 后端测试覆盖草稿过滤、历史数据兼容、非法发布状态校验和 Unicode 字符边界。
- 前端执行格式化、lint、类型检查和生产构建。

## 生产演进

- 当前配置以 JSON 整体覆盖保存，多管理员同时编辑时可能互相覆盖。后续应迁移到版本化 CRUD API，并通过 ETag 或版本号实现乐观并发控制。
- 增加 `created_by`、`updated_by`、`published_at` 和变更审计，满足生产环境追溯要求。
- 编辑已发布记录时创建 revision 或新 ID，避免用户已读状态继续绑定旧内容。
- Classic 前端渲染公告 Markdown 前需要统一净化 HTML，避免管理员内容形成存储型 XSS；Default 前端继续保留 DOMPurify 防护。
