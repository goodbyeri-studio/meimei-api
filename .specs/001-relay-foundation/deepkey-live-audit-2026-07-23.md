## 客户归属与计费日志

使用同一 `gpt` 上游渠道（渠道 10）完成了三次真实调用：

- 客户 A 首次请求记录为同一 `user_id` 和 `token_id`。
- 修改客户 A 的本地 Token 名称和 Key 值后再次请求，日志仍保持原 `user_id` 和 `token_id`，仅 `token_name` 更新。
- 客户 B 使用独立本地 Token 请求同一上游渠道，日志记录为不同的 `user_id` 和 `token_id`。
- 三次请求均记录 `channel_id`、`prompt_tokens`、`completion_tokens`、`quota` 和 `use_group`。
- 实测三次输入 Token 均为 28，输出 Token 分别为 17、15、17，对应费用额度分别为 85、77、85。

因此，客户费用归属依赖 Relay 本地 `user_id` 和 `token_id`，不依赖客户可修改的 Key 字符串，也不依赖多个客户共同映射的 DeepKey 上游 Key。
