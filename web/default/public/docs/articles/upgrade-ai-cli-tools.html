升级 Claude Code / Codex / Gemini CLI

三大主流 AI 编程 CLI 工具（Claude Code、OpenAI Codex CLI、Google Gemini CLI）迭代非常频繁，新版本会持续带来新功能、Bug 修复和性能改进。本文提供一条龙的 Bash 一键升级脚本，帮你保持工具始终处于最新版本。

一键升级脚本

将以下内容保存为 upgrade-ai-cli.sh：

#!/bin/bash

# Claude Code
npm i -g @anthropic-ai/claude-code

# Codex
npm i -g @openai/codex@latest

# Gemini CLI
#npm i -g @google/gemini-cli@latest
npm i -g @google/gemini-cli@nightly

赋予执行权限并运行：

chmod +x upgrade-ai-cli.sh
./upgrade-ai-cli.sh

分项说明

Claude Code 

npm i -g @anthropic-ai/claude-code
  • 包名：@anthropic-ai/claude-code
  • 不带版本号时默认安装最新稳定版
  • 升级后可运行 claude --version 验证

OpenAI Codex CLI 

npm i -g @openai/codex@latest
  • 包名：@openai/codex
  • @latest 显式指定安装最新发布版本
  • 升级后可运行 codex --version 验证

Google Gemini CLI 

# 稳定版
npm i -g @google/gemini-cli@latest

# Nightly 版（脚本默认使用这一行）
npm i -g @google/gemini-cli@nightly
  • 包名：@google/gemini-cli
  • 提供两个发布通道：@latest（稳定）和 @nightly（每日构建，功能更新快但可能不稳定）
  • 想要尝鲜新功能选 @nightly，追求稳定选 @latest

常见问题

Q：执行时提示权限不足（EACCES）怎么办？

macOS / Linux 下如果 npm 全局目录在系统路径，可能需要 sudo，但更推荐配置 npm 的全局目录到用户目录以避免权限问题：

mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH

Q：国内下载非常慢怎么办？

参考 NPM 中国加速配置指南 切换国内镜像源。

Q：如何确认是否升级成功？

分别运行：

claude --version
codex --version
gemini --version

对比输出版本号是否为最新。
