📋 使用建议

提示词编写原则

  1. 明确具体：详细描述需求，避免模糊表达
  2. 分步骤：复杂任务拆分为多个步骤
  3. 设定边界：明确什么要做，什么不要做
  4. 包含上下文：提供必要的背景信息
  5. 验证确认：重要步骤要求确认再继续

常用修饰词

  • "请先分析..."：要求 Claude 先理解再行动
  • "暂时不要..."：设定明确的边界
  • "每完成一步都要暂停"：控制执行节奏
  • "遵循项目现有..."：保持一致性
  • "如果需要...请告诉我"：主动沟通

效率提升技巧

  1. 使用 @ 引用文件：@src/components/Button.tsx
  2. 利用扩展思考：按两次 Shift+TAB 进入 PLAN 模式
  3. 合理清理上下文：使用 /clear 和 /compact
  4. 创建自定义命令：在 .claude/commands/ 目录下
  5. 批量操作：一次性处理多个相似任务
